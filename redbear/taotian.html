🎯
难点 1：意图识别不精准 高优先级
根因：用户表达多样性 + 上下文依赖 + 领域歧义
⌄
⚠️ 为什么无法完全解决
  • 自然语言天然存在歧义，同一问题可有数百种表达方式
  • 用户常用简短/方言/缩写表达，模型训练数据难以穷举覆盖
  • 多轮对话中历史上下文使意图动态变化，单轮识别无效
  • 电商细分场景（尺码/物流/售后）边界模糊，难以精确分类
✅ 改善方案
混合意图识别
规则 + 分类模型 + LLM 三层兜底，置信度投票机制
Few-shot 动态提示
实时检索相似历史成功案例注入 Prompt
意图澄清机制
置信度 < 阈值时主动反问，引导用户补充信息
持续微调
收集误识别样本，定期 SFT 微调领域模型
👻
难点 2：LLM 幻觉导致答错 高优先级
根因：模型参数化知识与实时数据脱节 + 生成性不可控
⌄
⚠️ 为什么无法完全解决
  • LLM 本质是概率模型，无法保证事实正确性，尤其检索结果不足时
  • RAG 召回的文档可能存在过期、矛盾、截断，模型仍会强行生成
  • 模型对不确定问题倾向给出自信答案（奉承性偏差）
  • 商品价格/库存/物流等实时数据无法完全注入静态知识库
✅ 改善方案
Grounding 约束
强制要求模型基于检索文档回答，不允许凭空生成
置信度熔断
检测回答与原文相似度，低于阈值触发转人工
实时 API 集成
价格/库存/物流直接调用淘天 OpenAPI，不经过 LLM
答案验证层
增加后处理校验，检测关键实体（价格/日期）合理性
🔄
难点 3：转人工时机判断 中优先级
根因：缺乏明确量化转接标准 + 用户情绪识别难
⌄
⚠️ 为什么无法完全解决
  • 转接过早浪费人工资源，转接过晚导致用户流失，边界难精确定义
  • 用户情绪（愤怒/紧急）在纯文字中难以准确识别
  • 同一问题不同用户有完全不同的处理预期
✅ 改善方案
多维度评分
综合置信度、重复次数、情绪分、问题复杂度触发转接
情绪识别模型
专门训练客服情感分类模型，实时监测会话情绪
用户主动触发
每轮对话提供显眼的「转人工」按钮，降低用户寻找成本
📚
难点 4：知识库覆盖率与时效性 中优先级
根因：电商数据量大、更新频繁，知识维护成本高
⌄
⚠️ 为什么无法完全解决
  • 商品 SKU 动态更新，知识库同步存在不可避免的延迟
  • FAQ 人工录入成本高，长尾问题覆盖严重不足
  • 多格式数据（图片/PDF/表格）解析质量参差不齐
✅ 改善方案
自动知识抽取
定时爬取商品详情页、评价、客服历史自动更新向量库
增量索引
支持实时增量更新，新上架商品 5 分钟内入库
知识图谱补充
构建商品-属性-FAQ 图谱，提升关联问题召回率
