03
核心技术方案
三大技术难点的落地路径，均已结合项目冷启动现实与100万预算约束进行优化设计。
难点 01 · 最高风险项
IRT 自适应算法——三段演进策略
算法
Phase 1 · 0–6月
规则引擎兜底
JLPT 官方分级预标注难度1–5级，EAP 简化估计 θ 值。答对3题连续升级，答错2题连续降级。此阶段算法可靠，承诺指标保守表述。
Phase 2 · 6–12月
混合过渡期
单题积累超 500 条作答数据后启动 EM 算法参数标定。高置信度题目切换 IRT 选题，低数据量题目继续规则兜底。
Phase 3 · 12月+
全量 IRT 切换
全部题目完成三参数标定（难度 b / 区分度 a / 猜测率 c），实现真正 CAT 计算机自适应测试，精度达到验收标准。
难点 02 · 差异化核心
记忆科学架构——三层用户记忆体系
LLM / mem0
⚡
短期记忆
本次测试会话状态
Redis · TTL 24h
🧠
情景记忆
历史答题模式归纳
向量 DB · 周期更新
📋
语义记忆
用户学习画像摘要
PostgreSQL · 长期
每次 LLM Agent 交互时，系统从三层记忆中检索相关上下文注入 Prompt，使 AI 能"记得"用户上次 N3 语法假定形持续出错、备考剩余 45 天、习惯晚间 10 点学习等个性化信息，规划质量远超通用方案。这是本项目相对竞品的核心技术护城河。
难点 03 · 数据资产
多维度用户历史数据记录
事件溯源
答题层
题目 ID · 作答选项 · 耗时（毫秒）· 修改次数 · 最终答案
会话层
测试类型 · 进入时间 · 中断次数 · 断点续答情况
能力层
每题后 θ 值估计 · 置信区间 · 等级变化轨迹
行为层
学习时间分布 · 连续学习天数 · 规划完成率
采用事件溯源（Event Sourcing）模式存储，任意时间点的用户状态均可回溯——既是算法优化的基础，也是后续 IRT 标定的核心数据资产。

高
算法工程师招聘延迟
IRT 方向算法工程师为稀缺资源，招聘周期 2–3 个月。应对：合同签订后第一周立即发布岗位，同步接触 2–3 名候选人；Phase 1 规则引擎期由后端工程师临时承接，降低招聘延迟影响。
中
IRT 冷启动精度争议
Phase 1 规则引擎期精度有限，客户可能误以为最终效果如此。应对：合同中分阶段写明验收指标（Phase 1 MAE ≤ 0.6，Phase 2+ MAE ≤ 0.5），并约定 IRT 全量切换里程碑时间点。
中
等保测评不通过需整改
首次等保测评通过率约 60–70%，不通过需整改复测，增加 2–4 周时间和约 2 万费用。应对：与开发同步启动等保差距评估，预留整改时间在项目周期内（Month 9–10 缓冲期），费用已含入风险预留。
低
范围蔓延
客户或总包方在项目执行中提出教师端、发音评测等二期功能。应对：合同附件中明确列出"本期范围外功能清单"，任何新增需求走书面变更单流程，按工作量单独报价，不影响本期利润。
