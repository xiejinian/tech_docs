🔧 实在智能产品体系速览
公司背景：浙江实在智能科技有限公司，国家高新技术企业，2025年3月完成近2亿元C轮融资。核心定位：AI × RPA 超自动化，自研 TARS 大模型 + ISSUT 屏幕语义理解技术，是国内信创RPA生态覆盖最全面的厂商之一。
🧠 实在Agent + 企业大脑
核心通用智能体平台，2025年3月最新升级版本。包含：
  • 专属知识库RAG：将企业文档/数据沉淀为可检索知识资产
  • 业务流程数字孪生：可视化智能体画布，拖拽搭建自动化流程
  • 多模型接入：TARS / DeepSeek / 智谱AI / GPT 按需切换
  • 私有化部署：支持账号授权、角色管理、国产化适配
知识问答 RAG检索 私有化 信创
📄 IDP 智能文档审阅（含Chat-IDP）
基于大模型的文档理解平台，核心能力：

  • 关键信息抽取、内容风险审查、智能归档
  • Chat-IDP：文档级对话问答，划选原文即可提问
  • 从文档输入到解析完成仅需数秒
文档解析 字段抽取 OCR 合规审核

📚 场景一：智能知识问答 · 实在智能方案
📊 实在智能方案适配度：极高（95分）— 开箱即用，重点在知识治理
核心判断：企业大脑的知识库RAG + IDP文档审阅，完整覆盖"知识问答场景"的全部技术需求。81条问答样本中涵盖的标准规范、作业指导书、会议纪要等文档类型，均在IDP的原生支持范围内。这个场景几乎不需要额外自研，重点工作转为知识治理与系统配置。
实在产品 → 功能对应
  • ▸IDP平台：承接Word/PDF/扫描件解析，包括OCR，秒级完成
  • ▸企业大脑RAG：构建合肥高铁专属知识库，自动向量化、分类存储
  • ▸实在RPA：实现文件夹自动监听，新增文档自动触发入库流程，无人干预
  • ▸Chat-IDP：支持自然语言问答，多轮对话，划选原文追问
  • ▸权限管控：企业大脑管理平台原生支持知识库级别权限设置
飞凡科技定制工作
  • ▸知识分类配置：按合肥高铁实际业务建立"标准规范/红头文件/作业指导书"分类体系
  • ▸铁路专有名词词典：6C/2C/3C/5C/HMVP等专业缩写的同义词扩展配置
  • ▸切片策略调优：按铁路规范文档特点（条款编号体系）定制切片逻辑
  • ▸扫描件质检：需要对10000份文档抽样，确认OCR质量，异常文档需人工处理
  • ▸FAQ库初始化：将81条样本题库导入FAQ，作为高频问答优先响应
工期变化：原自研方案知识库构建需4周（含框架搭建），使用实在产品后压缩至2周（仅含配置+知识治理+调优），节省约2周开发时间。
📝 场景二：工作票智能审核 · 实在智能方案
⚡ 实在产品适配度：中等（70分）— IDP承接解析，规则引擎需定制叠加
技术路径：工作票审核分两层——第一层由IDP承接（字段解析、格式识别、结构化提取），第二层由飞凡科技定制规则引擎（基于提取的结构化数据进行合规校验），两层通过API串联。实在产品原生不提供规则引擎，这是定制开发的核心部分。
⚡ 实在产品原生覆盖 · IDP自动识别票种、解析版式
· 关键字段抽取（票号/时间/地点/人员）
· 支持Word/Excel固定模板解析
· 实在RPA批量上传工作票触发审核
→
🔧 飞凡科技定制叠加 · 规则引擎：5大类审核规则代码化
· 安全等级数据库对接
· 违规字段精准标注逻辑
· 整改建议生成（Prompt工程）
· 审核报告模板渲染
实在产品承接的部分（省工）
  • ▸IDP模板适配：3类工作票格式识别，无需从零开发解析器
  • ▸字段抽取：时间、地点、人员等结构化信息自动提取
  • ▸Chat-IDP：工作票内容问答（辅助人工审核时快速查询原文）
  • ▸TARS/DeepSeek："安全措施上下文无冲突"判断，直接调用LLM语义理解
必须自研的定制部分（仍需工作）
  • ▸规则引擎代码：5条审核规则转化为可执行判断逻辑
  • ▸数据库对接：安全等级人员数据库查询接口
  • ▸违规报告生成：审核结果结构化展示界面
  • ▸地点合规校验：区段/支柱号匹配需路网数据库，实在产品无法覆盖
⚠️ 不变的核心警示：即使使用实在产品，"安全措施上下文无冲突"语义判断的误判风险依然存在。合同中必须明确本系统为辅助审核工具，最终责任由人工承担，这一条款与技术方案无关，是保护飞凡科技的法律边界。

R4
客户对"第三方产品"的采购合规问题

🟠 中
📌 应对措施 建议将实在智能产品授权费纳入飞凡总合同报价（而非客户单独采购），减少客户侧采购流程；方案中强调实在智能的信创认证资质，降低合规审查压力。
R5
问答准确率验收（不变风险）
使用实在产品的RAG引擎同样无法保证答案逐字匹配，验收标准争议风险不因选型改变而消除。
