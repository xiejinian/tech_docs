Project Phoenix · 12-Month Opportunity Engine

个人高端职位获取与飞凡商业机会回流自动化方案

以个人职业跃迁为主线，以飞凡科技业务机会为副线，通过 LinkedIn、Boss、猎聘、Upwork、专家咨询平台与内容运营，建立可自动化、可评分、可跟进、可复盘的机会增长系统。

身份定位：CTO / AI Technology Executive 周期：未来12个月 目标：职位 + 项目 + 资源网络 交付对象：Codex 自动化实现

一、战略定位

核心不是“找工作”，而是进入更高价值的平台，并把职位、人脉、客户和项目资源转化为飞凡科技的长期商业入口。

01

高端职位

锁定 CTO、VP Engineering、Director of Engineering、Head of AI 等岗位。

02

海外机会

重点关注日本、新加坡、泰国、马来西亚等东亚与东南亚市场。

03

个人品牌

从“Java架构师”升级为“AI技术高管 / 企业AI顾问”。

04

飞凡回流

将职位沟通、行业人脉、项目咨询转化为飞凡潜在客户与合作伙伴。

六、自动化Agent架构

以下模块可交给 Codex 按服务化方式实现，先MVP，后续接入浏览器自动化、邮件、飞书/企微、Notion或自建CRM。

Job Hunter Agent

抓取 LinkedIn、Boss、猎聘职位，提取岗位、公司、行业、地点、薪资、JD、联系人。

Project Hunter Agent

抓取 Upwork、Toptal、Catalant 项目机会，识别AI顾问、CTO顾问、企业AI项目。

Scoring Agent

基于岗位层级、公司质量、AI相关性、国际化程度、飞凡协同价值进行评分。

Outreach Agent

生成不同平台的打招呼、跟进、面试回应、项目Proposal，支持人工审核后发送。

CRM Agent

管理线索状态、联系人、沟通记录、下一步动作、提醒和周报。

Content Agent

根据沟通与项目沉淀，生成LinkedIn文章、X短帖、中文长文、案例总结。

七、机会评分模型

所有机会统一进入100分模型，避免凭感觉投入时间。

岗位层级25CTO/VP/Director/Head 最高；普通架构师、开发岗降权。
公司质量20融资阶段、营收、团队规模、客户质量、行业位置。
国际化程度15海外团队、跨境业务、东南亚/日本市场优先。
AI相关性15Agent、RAG、客服、外呼、企业AI平台相关性越强越高。
飞凡协同价值15是否可能导入客户、项目、渠道、合作伙伴。
薪资/回报10薪资、股权、顾问费、项目预算。

暂不投入，归档作为数据参考。

九、分平台SOP

不同平台的动作不同，但必须统一回流到CRM。

LinkedIn
每日抓取高管职位与目标公司动态；每周新增50个目标联系人；对S/A级机会发送个性化连接请求；每周发布1篇英文AI落地/CTO管理内容。
Boss直聘
每日搜索CTO、技术总监、AI负责人、Agent、RAG等关键词；对招聘背后的项目需求进行挖掘；符合条件的需求转为飞凡商机。
猎聘
每周批量筛选高薪技术管理岗；重点关注猎头、企业HR和隐性职位；适合作为高端职位补充池。
Upwork/Toptal
每日筛选AI顾问、企业AI、技术尽调、Fractional CTO项目；生成英文Proposal；人工确认后提交。
专家咨询
完善VISASQ/GLG/AlphaSights资料；突出中国AI应用、企业软件、智能客服、AI外呼、RAG、Agent落地经验。
内容运营
每周沉淀1篇英文LinkedIn长文、2-3条X短帖、1篇中文复盘；内容主题围绕AI落地、CTO管理、创业团队技术决策。

十、Codex工程实现指令

以下内容可直接作为Codex任务说明。建议先做本地MVP，再逐步接入真实平台。

# Codex Task: Build Project Phoenix MVP ## Goal Build a local-first automation system for personal career opportunity discovery and Feifan business lead backflow. The system should collect opportunities from multiple platforms, score them, generate outreach drafts, and manage follow-up actions in a CRM-style dashboard. ## Tech Stack Recommendation - Backend: Python FastAPI - Database: SQLite for MVP, PostgreSQL-ready schema - Frontend: React + Vite + Tailwind or simple HTML dashboard - LLM: configurable provider interface, default OpenAI-compatible API - Scheduler: APScheduler or cron - Export: CSV / Markdown / JSON ## Core Modules 1. opportunity_ingestion - Input: manually uploaded CSV/JSON first; later add browser automation connectors. - Supported sources: linkedin, boss, liepin, upwork, toptal, catalant, expert_network. - Normalize each item into Opportunity schema. 2. opportunity_scoring - Implement 100-point scoring model: job_level: 25 company_quality: 20 internationalization: 15 ai_relevance: 15 feifan_synergy: 15 compensation_or_budget: 10 - Output score_total and score_breakdown. - Classify as S/A/B/C. 3. outreach_generation - Generate platform-specific outreach draft. - Templates: LinkedIn connection note <= 300 chars Boss greeting <= 140 Chinese chars Upwork proposal <= 1200 English chars Follow-up message <= 300 chars - Must require manual approval before sending. 4. crm_dashboard - Views: New Opportunities S/A Priority Queue Follow-up Today Feifan Synergy Leads Archived - Fields: source, type, title, company, location, score, stage, next_action, next_action_date. 5. reporting - Generate weekly Markdown report: total opportunities collected S/A/B/C distribution top 10 opportunities Feifan-related leads pending follow-ups recommended actions next week ## Safety and Compliance Requirements - Do not auto-send messages in MVP. - Do not scrape aggressively. - Respect platform terms; prefer manual export or user-authorized browser automation. - Store credentials outside code using environment variables. - Keep all outbound messages in draft state until manually approved. ## MVP Acceptance Criteria - User can import a CSV of opportunities. - System normalizes records into Opportunity schema. - System scores each opportunity. - System generates outreach drafts. - Dashboard shows prioritized opportunities. - Weekly report can be exported as Markdown. ## Later Enhancements - Browser extension for one-click save from LinkedIn/Boss. - Gmail/Calendar integration for meeting and follow-up tracking. - Notion/Feishu sync. - Company enrichment via public sources. - Content Agent for LinkedIn/X post drafting. - Multi-language resume and profile optimizer.

十二、关键风险与边界

个人职位获取与公司商机回流之间要保持合规边界。

利益冲突

若进入目标公司任职，必须区分个人职责与飞凡合作机会，避免未经授权的资源转移。

平台规则

自动化应优先采用人工导入、浏览器辅助、草稿生成，避免违规批量抓取和批量发送。

个人定位

不要降维到普通开发岗位，应持续强化CTO、AI技术高管、企业AI顾问定位。
