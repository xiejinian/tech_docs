3 Win11 可靠性保障体系

目标：99%+ 安装成功率
🔍

前置依赖矩阵

  • ✓ 系统版本 ≥ Windows 11 21H2 (Build 22000)
  • ✓ .NET Framework 4.8.1+ (WinSW依赖)
  • ✓ 磁盘空间 ≥ 1GB 可用
  • ✓ 内存 ≥ 4GB（Node运行时需求）
不满足条件时优雅降级，提供清晰的解决指引
🛡️

安全软件兼容性

  • ✓ 代码签名证书（EV证书最佳）
  • ✓ Windows Defender SmartScreen 白名单申请
  • ✓ 安装包SHA256校验和公开提供
  • ✓ 杀毒软件提交（360、火绒、卡巴斯基）
避免误报导致的安装阻断，提升消费者信任度
🔄

故障恢复机制

  • ✓ 事务性安装：失败自动回滚
  • ✓ 端口占用检测（18789冲突处理）
  • ✓ 服务启动超时保护（30s重试机制）
  • ✓ 详细日志输出（%Temp%\YourProduct_Install.log）
消费者场景需避免技术黑盒，提供可理解的错误提示

离线场景下的更新机制

# 策略A：增量离线包（推荐）
client → 检测更新 → 下载.diff补丁 → 本地合并
# 策略B：全量替换（简单）
client → 提示新版本 → 下载完整.exe → 调用自更新
# 策略C：混合模式（消费者友好）
WiFi时自动增量更新 / 离线时提示手动下载
考虑到消费者可能长期离线，需支持"离线更新包"手动导入机制
