MODULE A

合同审核辅助Agent · 可行性拆解

高风险项存在偏差则一票否决，需标红高亮；中风险项为合规提示，无否决权。技术路径标注了该项在实在智能平台上主要依赖的能力类型。

3.1 单价合规校验（房屋租赁）
中等难度
OCR文档理解 + 规则比对

合同单价 vs 评估报告评估价数值比对。评估报告格式不统一（多为扫描件/PDF），字段位置不固定，需Agent语义抽取而非固定模板。

风险：评估报告抽取准确率直接决定该项能否达到"强制校验"要求的可靠性，建议POC验证。
3.2 多文件信息一致性校验
较高难度
Agent语义理解为主

合同、业务单据、会议纪要、成交通知书四类异构文档的交易内容/金额交叉核对。会议纪要属自由文本，规则引擎难以覆盖，需LLM级语义匹配。

风险：语义比对存在误判率，国企风控场景容错率低，建议设人工复核兜底，不承诺100%准确。
3.3 采购合同立项内容专项校验
中等难度
结构化字段比对

项目名称/类型/甲乙方/时间与立项内容匹配；金额不超立项且大小写一致；付款比例一致。以RPA规则引擎为主，前提是立项数据可靠抽取。

依赖：需先完成立项审批单的结构化/半结构化抽取（见模块B）。
3.4 流程时间逻辑顺序校验
低难度
RPA规则引擎（原生能力）

立项→合同发起→签订→履约四个时间节点先后顺序判断。时间字段抽取成熟后逻辑判断非常成熟，是全项目中确定性最高的一项。

3.5 审核流程完整性校验
中等难度
RPA存在性检测 + 截图OCR

必备材料齐全性检测（律师意见邮件截图、采购立项文件、租赁三类配套文件）。"律师提出修改意见需同步上传修订版"这类条件判断需要语义理解而非纯规则。

风险：邮件截图OCR准确性依赖图片清晰度，建议限定截图格式规范。
4.1-① 工作量与单价匹配校验
较高难度
依赖行业基准库

判断工作量与单价是否"合理"需要外部基准数据支撑，Agent本身无法独立定义合理区间，除非客户提供或协助建设基准表。

风险：基准库现状"未知需专项调研"，是本次估算中不确定性最大的一项，建议列为二期或降级为"异常值提示"。
4.1-② 违约责任权责校验
较高难度
LLM语义分析 + 范本比对

判断双方违约责任是否权责对等、公平合规，属于法律语义主观判断，AI只能做条款偏离度提示，不能替代终审。

合规提醒：SOW中需明确验收标准为"提示覆盖率"而非"判断准确率"，避免开放式责任承诺。
4.1-③ 付款比例合规校验
中等难度
结构化字段比对

比对合同付款比例与招标文件备案标准是否一致，抽取+比对逻辑成熟，前提是招标备案数据可获取。

MODULE B

采购立项审批Agent · 可行性拆解

含单据字段核验、文档一致性校验，以及每月定时执行一次的台账自动维护。

采购类型校验
低难度
Agent文本分类

从项目概述文本智能判定服务类/工程类/货物类，与单据填写类型核对。是较成熟的LLM文本分类任务。

关联交易校验
较高难度
依赖关联企业名单库

从项目概况抽取涉事企业名称，匹配内置关联企业名单判定关联交易。技术抽取本身不难，核心瓶颈是名单库是否已电子化、是否需要持续维护更新机制。

风险：关联企业名单现状未知，若客户无电子化名单，需新增名单维护流程与责任归属约定。
是否上会判定
低难度
RPA规则引擎（原生能力）

采购金额≥60万自动标记需上会，纯阈值规则判断，确定性最高。

采购方式校验
较高难度
依赖类别对照表

结合"采购类型+采购金额"双维度匹配"采购项目类别与使用范围对照表"，校验采购方式合规性。同样依赖对照表电子化程度。

风险：与工作量单价基准表同属Phase 0调研的核心待办项。
文档一致性 · 强制一致项
低难度
RPA规则引擎

项目名称须完全一字不差匹配，字符串精确比对，非常成熟。

文档一致性 · 宽松一致项
中等难度
Agent语义相似度

项目概述语义/核心信息一致即可，允许文字表述小幅差异，需要LLM语义相似度判断而非精确匹配。

采购台账自动生成（月度）
低难度
RPA批处理（原生能力）

全量采购立项单据汇总为Excel单行记录，定时批量导出。技术成熟，质量取决于前序字段抽取的准确性。

合同台账自动生成（月度）
低难度
RPA批处理（原生能力）

采购立项单+合同附件信息汇总为Excel单行记录，与采购台账同批次交付。

RISK REGISTER

关键风险登记册

内部评估专用，用于报价阶段的风险定价与合同条款设计参考。

风险项等级影响应对建议
非结构化文档抽取准确率（评估报告 / 会议纪要 / 邮件截图OCR） 高 高风险强制校验项（单价合规、多文件一致性）的可靠性直接受抽取准确率影响，误判可能导致合同被错误放行或错误拦截 Phase 0阶段用客户真实样本做抽取准确率POC；上线后保留人工复核环节，不承诺100%自动判定
规则库/基准表/关联企业名单现状不明 高 直接影响Phase 3工作量与交付时间，估算区间±约20%的不确定性主要来自此项 Phase 0作为单独立项/计费阶段，产出《数据接口确认书》+《规则库建设范围确认书》，作为正式报价依据
主观判断类校验的AI能力边界（权责对等性、单价合理性） 中 客户可能期望"自动判定"，但AI只能做"提示级"辅助，存在预期落差与验收争议风险 SOW明确此类交付物为"风险提示清单"而非"合规判定"，验收标准约定为提示覆盖率而非判断准确率
钉钉API权限与私聊/群聊推送依赖 中 集成开发进度受客户IT侧配合程度影响；群聊场景还涉及"应推给谁"的路由规则确认 项目启动前确认表单API、代办任务转派、私聊/群机器人推送三类权限已开通
国企风控场景对误判容忍度低 中 全自动化放行可能引发合规追责风险 方案定位明确为"审核辅助"而非"替代终审"，所有高风险判定保留人工确认环节
ROADMAP

分期实施建议

按技术确定性从高到低排期，优先上线规则引擎类校验建立客户信任，再逐步引入文档理解与规则库依赖类能力。

PHASE 0

专项调研（建议单独计费）

  • 规则库/基准表/关联企业名单现状盘点
  • 合同、评估报告、立项单据样本抽样，评估OCR可行性
  • 钉钉API权限确认
产出：数据接口确认书 + 规则库建设范围确认书
PHASE 1

规则引擎类校验上线

  • 时间逻辑顺序校验
  • 上会判定 / 强制一致项 / 金额大小写一致
  • 钉钉表单对接 + 私聊/群聊双通道推送
确定性最高，优先建立客户信任
PHASE 2

文档理解类校验

  • 合同全量文本抽取
  • 评估报告 / 会议纪要语义比对
  • 邮件截图OCR识别
核心工作量所在，需样本调优周期
PHASE 3

规则库依赖类 + 台账

  • 关联交易 / 采购方式校验
  • 采购台账 / 合同台账自动生成
前提：Phase 0确认规则库已就绪
PHASE 4

主观判断辅助项

  • 违约责任权责提示
  • 工作量单价合理性提示
建议作为"增强项"交付，不作硬性验收承诺

OPEN ITEMS

关键假设与Phase 0待确认事项

以下为本次售前区间估算所依赖的假设，正式报价前需在Phase 0中逐项确认，任一假设偏差都会带来估算区间外的调整。

1
数据链路：假设合同/立项数据需同时对接钉钉表单结构化字段与附件全文抽取（OCR/语义解析），若客户系统已有结构化字段沉淀可直接调用，Phase 2工作量可下调约15%–20%。
2
规则库现状：关联企业名单、采购类别对照表、工作量单价基准表现状未知，本次估算按"需协助新建"的较高工作量口径估算，若客户已有电子化数据，Phase 3工作量可下调约30%。
3
验收边界：违约责任权责校验、工作量单价合理性校验属主观判断类，建议在SOW中明确为"辅助提示"而非"合规判定"，避免无限责任承诺。
4
样本可得性：Phase 2的抽取准确率POC需要客户提供不少于20份真实合同、评估报告、会议纪要样本，样本量不足会延长调优周期。
5
推送路由规则：Agent分析结果需推送给"对应人或群"，需在Phase 0明确路由判断逻辑（何时推个人、何时推群、群内@谁），以及群机器人的权限申请方式，避免上线后出现误推/漏推。
