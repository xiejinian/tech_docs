01 · Executive Summary

方案 概述

在不重构核心架构的前提下，为现有 FreeSWITCH 呼叫中心系统增加自然、低延迟的智能打断能力。

🎯
核心目标
在 TTS 播报过程中实时判断用户是否具备"有效发言意图"，在 150ms 内完成打断决策并切换至 ASR 识别流程，显著提升交互自然度与用户满意度。
🏗️
方案核心
在 ASR 模块内集成打断决策引擎（IDE），融合 VAD、声学特征、语义意图、对话上下文四维信号加权评分，统一对外发布打断事件。
📐
架构原则
ASR 模块为打断信号的唯一权威来源，TTS 模块与业务层保持被动执行者角色，各模块职责清晰、可独立迭代升级，避免跨模块状态耦合。
📈
预期收益
打断准确率目标 ≥ 90%，误触发率 ≤ 5%，通话自然度评分提升 30%+，支持投诉、支付等差异化业务场景动态配置。
关键设计决策：打断的本质是"用户输入信号"与"系统输出行为"的仲裁。ASR 模块负责裁决"用户想说话了吗"，业务层负责策略决定"现在该不该让他说"，TTS 模块负责执行停播。三层职责清晰分离，是本方案最核心的架构决策。

04 · Scoring Model

四维加权评分模型

多维信号融合输出 0–1 打断置信度，超过阈值则触发打断。

打断分数 = 0.30 × VAD分 + 0.25 × 声学特征分 + 0.30 × 语义意图分 + 0.15 × 上下文分
综合得分 ≥ 0.65 触发打断 · 精确关键词命中则直接进入快速通道
Dimension 01
VAD 语音活动检测
WebRTC VAD 一级过滤，延迟 <1ms，作为信号门控层
Silero VAD 二级确认，精度更高，缓存 100ms 后判断
最低有效语音持续时长 120ms，过滤短暂噪音
双层架构兼顾速度与精度，置信度要求 ≥ 0.75
Dimension 02
声学特征分析
HNR（谐波噪声比）：人声 >10dB，噪音 <5dB，最强判据
基频 F0：人声集中 80–800Hz，超范围则降分
频谱质心：语音集中在 300–3000Hz 带内
能量斜率：人声有明显上升沿，区别于稳态噪音
Dimension 03
语义意图识别
流式 ASR 实时转写，不等完整句子即获取部分结果
关键词规则库优先匹配（<1ms）：停止 / 确认 / 异议类
本地轻量意图分类模型兜底，约 10MB，延迟 <50ms
精确匹配打断词时直接触发快速通道，跳过加权评分
Dimension 04
对话上下文评分
TTS 播报超 3 秒：用户更可能想打断，动态加分
当前句为长句（>15s）：适当降低打断阈值
用户历史打断率高：倾向于认为是有效打断
近期发生误打断：触发冷却，提升本次阈值
05 · State Machine

打断决策状态机

从 TTS 播报到打断执行的完整状态流转路径。

PLAYING
TTS 播报中 · 静默监听用户输入
VAD 触发 · 检测到 >120ms 有效语音
↓
ANALYZING
多维特征提取 50–200ms · TTS 并行继续播报
评分计算完毕
分数 < 阈值
↓
CONTINUE
继续播报 · 重置分析窗口
分数 ≥ 0.65 或快速通道
↓
INTERRUPT !
停止 TTS · 激活 ASR
恢复预缓冲音频
冷却机制：打断后 800ms 内不再触发 · 误触发后冷却 2000ms · 预缓冲 300ms 音频防止截头丢失
06 · Module Boundary

ASR 模块职责边界

ASR 收口的核心原则：负责裁决，不负责策略；发布事件，不执行动作。

FreeSWITCH 媒体层
持续提供原始 RTP 音频流
接收并执行 ESL 控制命令
uuid_break 停止播报
不参与任何打断判断逻辑
不持有任何 AI 推理状态
✅ ASR 模块（统一收口）
VAD 检测（原有能力扩展）
声学特征分析（区分人声/噪音）
打断置信度加权评分
预缓冲音频管理（防截头）
统一对外发布打断事件
❌ 不持有 TTS 播报状态
❌ 不感知业务场景差异
❌ 不直接执行停播操作
业务层 & TTS
订阅 ASR 发布的打断事件
根据业务场景决定是否响应
投诉场景降低响应阈值
支付场景提高响应阈值
TTS 收到指令后执行停播
不参与任何语音分析判断
关键陷阱预警：ASR 模块不应感知"当前 TTS 是否在播报"。ASR 持续运行，打断信号随时产出；是否响应该信号是业务层的策略权。若 ASR 依赖 TTS 状态，TTS 故障将导致 ASR 打断能力同步失效，形成难以排查的跨模块 bug。

09 · Configuration

推荐阈值配置

可按业务场景动态调整，以下为推荐基准配置。

# 智能打断策略配置 · interrupt_policy.yaml

trigger:
vad_duration_min: 120ms # 最小有效语音持续时长
vad_confidence: 0.75 # VAD 置信度要求

scoring:
weights:
vad: 0.30
acoustic: 0.25
semantic: 0.30
context: 0.15
threshold: 0.65 # 基准打断阈值

fast_track: # 直接打断，跳过加权评分
keyword_exact_match: true
semantic_score_min: 0.90

scene_overrides: # 业务场景阈值偏移
complaint: -0.10 # 投诉场景：更易打断
payment: +0.10 # 支付确认：更难打断
greeting: +0.05 # 开场白：略微保守

noise_suppression:
snr_threshold: 8dB # 信噪比低于此值时提高阈值
use_ns_model: true

cooldown:
after_interrupt: 800ms
after_false_positive: 2000ms
pre_buffer: 300ms # 预缓冲防截头
10 · Implementation Roadmap

分阶段实施路线图
