🔄 关键数据流

核心场景的启动与交互流程

🚀 应用启动流程

  1. SmartDisplayApp.onCreate() → 初始化 DeviceInfo, UserManager, Sentry
  2. IdleHandler 触发 → UpdateManager 观察 + 全局串口输入注册
  3. MainActivity → 显示 HelloScreen → 进入 MainContent
  4. AppInitManager.start() 三阶段初始化
    A: BootstrapSync
    B: BLE, WiFi, KeepAlive, Auth
    C: Customization, Precreator
  5. 已登录 → CalendarWebScreen 待机主屏 | 未登录 → WelcomeActivity 引导

🗣️ AI 语音对话流程

  1. 唤醒词 IFlytekKws 检测到关键词 → 回调触发
  2. 启动 ChatActivity → ChatViewModel 初始化
  3. 创建会话 → 服务器返回 ChatConfig (chain / realtime)
  4. Chain 模式: WebSocket 通道 + STT → Agent → TTS
  5. Realtime 模式: WebRTC PeerConnection 双向音频
  6. 音频数据流: AudioRecord → Opus 编码 → 上行 | 下行 → AudioPlayer 播放
  7. 静音检测 30s 无交互 → 自动结束会话

📱 BLE 配网流程

  1. TV 启动 BLE 广播 → 生成 QR 码 (含设备公钥)
  2. 手机扫码 → BLE GATT 连接 → RX_CHAR / TX_CHAR 双特征
  3. X25519 ECDH 密钥交换 + Ed25519 签名认证
  4. AES-256-GCM 加密通道建立 → HandshakeCompleted
  5. 手机发送 WifiCredentials → TV 连接 Wi-Fi
  6. 手机发送 LoginToken → TV 登录认证
  7. 支持: 退出登录, 版本更新, 设备状态查询

📋 会议模式流程

  1. 语音指令触发 → 启动 MeetingActivity
  2. TongYiTingWuService 初始化通义听悟
  3. 实时录音 → 语音识别 → 逐句回调 onSentence
  4. 多人说话分辨 (speaker) → UI 实时展示
  5. 关键词检测 → 自动结束 | 超时自动结束
  6. 会议纪要生成 → 结束对话弹窗

📦 OTA 更新流程

  1. KeepAlive 长连接 → 收到 update 推送
  2. UpdateManager 检查版本 → 下载 APK + SHA256 校验
  3. PackageInstaller 静默安装 (v1 + v2 签名)
  4. setDontKillApp(true) → 不杀进程更新
  5. 安装成功 → forceKillSelfForUpgrade() 重启
  6. 安装失败 → 回退状态, 继续当前版本

🌐 网络状态管理

  1. WifiConnectionService 监听 Wi-Fi 连接状态
  2. InternetProbeManager 多端点探测公网连通性
    Google 204 / Apple / MSFT / 自建 /health
  3. NetState 聚合: hasNetwork + hasInternet + hasBackend
  4. isOnlineFlow → UI 层响应式绑定
  5. BLE 蓝牙查询接口 → getCurrentNetworkStatusJson()

📦 核心模块详解

每个模块的职责和关键实现

📺

MainActivity

.MainActivity

应用主入口, 作为 Launcher + Home。包含 HelloScreen 开屏 → CalendarWebScreen 待机页 → 唤醒词监听与 AI 对话入口。管理引导流程与更新拦截。

🗣️

ChatActivity

.ChatActivity

AI 语音对话界面，基于 ChatViewModel 管理状态。支持初始化动画、语音音量可视化、AI 字幕、搜索图片展示、音量/亮度调节 HUD。

📋

MeetingActivity

.MeetingActivity

会议纪要模式。接入通义听悟 TongYiTingWuService, 实时语音转写, 多人话者分辨, 支持关键词自动结束与超时保护。

🌐

WelcomeActivity

.WelcomeActivity

新设备引导页面。显示 BLE QR 码、BLE 连接状态、Wi-Fi 配网进度、登录状态、OTA 更新状态。全流程可视化引导。

🧠

ChatViewModel

.chat.ChatViewModel

对话状态中心。管理 initializing、userSpeaking、aiThinking、aiSpeaking 等 StateFlow, 驱动 AIScreen Compose UI 响应式渲染。

⛓️

ChainChatService

.chain.ChainChatService

链式对话引擎。WebSocket 通道, STT→Agent→TTS 流水线。支持 Opus 编码, 自由打断, 静音超时检测, AI 回复超时。

🔴

RealtimeChat

.chat.RealtimeChat

基于 WebRTC 的实时对话。PeerConnectionFactory + JavaAudioDeviceModule, 支持 SDP 交换, ICE 候选, 双向音频流。

📡

AppBleService

.ble.AppBleService

BLE GATT Server 实现。双特征通道 (RX 0xA111 / TX 0xA112), 分包重组, 认证加密, 支持 Wi-Fi 配网、登录、更新等指令。

🔐

CryptoService

.ble.CryptoService

基于 BouncyCastle 的加密服务。X25519 ECDH 密钥交换 + Ed25519 身份签名 + AES-256-GCM 数据加密, 兼容 Android 12。

🎤

WakeWordDetector

.utils.IFlytekKws

讯飞离线唤醒词检测。后台线程 AudioRecord 采集, 支持从后端动态获取唤醒词配置, SDK 初始化后驻留监听。

🌍

NetState

.network.NetState

网络状态聚合中心。组合 Wi-Fi IP 检测 + 公网探测 + 后端长连接状态, 提供 hasNetworkFlow / hasInternetFlow / isOnlineFlow。

🔄

KeepAliveManager

.network.KeepAliveManager

WebSocket 长连接管理。自动重连, 接收服务端推送 (壁纸更新、AI 配置变更、远程指令、OTA 触发、工厂测试)。

📦

UpdateManager

.update.UpdateManager

OTA 更新管理器。APK 下载 + SHA256 校验 + PackageInstaller 静默安装 + DontKillApp 无缝更新 + 失败回退。

🔌

MySerialPort

.serialport.MySerialPort

串口通信模块。8 字节帧协议 (0x55AA 帧头), 支持亮度/音量读写, 按键事件 (MENU/UP/DOWN), 请求-响应匹配。

📝

AppLog + Sentry

.log.AppLog

统一日志框架。i/d/w/e 级别, 同时写 Logcat + 本地文件 + Sentry 上报。错误级别自动 captureException, 支持节流日志。

👤

UserManager

.user.UserManager

用户状态管理。userFlow 响应式, 支持登录/登出、Token 刷新, 通过 BLE 或直接 API 完成认证。

🎨

CustomizationManager

.user.CustomizationManager

用户自定义管理。壁纸类型 (默认/自定义)、布局类型 (默认/相框)、壁纸下载与版本管理、服务端推送同步。

🚪

ExitCoordinator

.exit.ExitCoordinator

统一退出协调器。处理 MENU 键事件, AI 对话中首击打断语音/次击弹出退出确认, AutoCloseDialog 超时自动关闭。

⚡

AppInitManager

.init.AppInitManager

分阶段初始化编排器。Stage A: 核心同步 → Stage B: BLE/WiFi/Auth → Stage C: 延迟加载。支持 stop 优雅关闭。

🔗

BackendServer

.backend.BackendServer

后端 API 客户端。对话创建、WebRTC 会话、Web 搜索、配额查询、设备上报, 基于 OkHttp + Fastjson2 异步调用。

🏭

FactoryTestManager

.factorytest.FactoryTestManager

工厂测试管理。远程指令: kill_self、ANR 阻塞测试、音频录制证明、亮度/音量读写测试, 结果回报后端。

🌤️

WeatherRepository

.web.weather.WeatherRepository

天气数据源。通过 JavaScript Bridge 与 WebView 中的日历页面交互, 提供天气 UI 数据模型。

🚀 快速开始

从克隆到运行的完整步骤

1

克隆仓库

git clone https://github.com/sihai-org/SmartDisplay.git cd SmartDisplay
2

配置 local.properties

在项目根目录创建 local.properties 文件，填入签名和环境配置：

# Android SDK 路径 sdk.dir=/path/to/Android/sdk # 签名配置 RELEASE_STORE_FILE=../keys/release.keystore RELEASE_STORE_PASSWORD=your_store_password RELEASE_KEY_ALIAS=your_key_alias RELEASE_KEY_PASSWORD=your_key_password # Sentry DSN (可选) SENTRY_DSN=https://xxx@sentry.io/yyy
3

放置本地 AAR 库

将以下 AAR 文件放入 app/libs/ 目录：

app/libs/ ├── opus.aar # Opus 音频编解码 ├── AIKit.aar # 讯飞 AI 语音套件 └── nuisdk-release.aar # NUI 语音 SDK
4

构建并运行

选择目标 Flavor 和 Build Type 进行构建：

# 本地开发 Debug 版本 ./gradlew assembleLocalDebug # Staging 测试版本 ./gradlew assembleStagingDebug # 生产 Release 版本 ./gradlew assembleProdRelease # 安装到设备 adb install -r app/build/outputs/apk/local/debug/app-local-debug.apk
5

设备配网

首次启动进入 WelcomeActivity → 使用手机 App 扫描屏幕上的 QR 码 → 通过 BLE 完成 Wi-Fi 配网和账号登录 → 进入主界面开始使用。

📡 通信协议说明

设备与外部系统之间的通信方式

🔵 BLE GATT Protocol

  1. Service UUID: 0000A100-...
  2. RX Char (手机→TV): 0000A111-...
  3. TX Char (TV→手机): 0000A112-...
  4. 认证: X25519 ECDH + Ed25519 + AES-256-GCM
  5. 指令: handshake / wifi.config / login / logout / update

🟢 串口帧协议

  1. 帧头: 0x55 0xAA (2 bytes)
  2. 功能码: 1 byte (亮度/音量/按键)
  3. 数据域: 4 bytes (data + requestId)
  4. 校验和: 1 byte (累加和 & 0xFF)
  5. 总帧长: 8 bytes 固定长度

🟡 WebSocket 长连接

  1. 地址: wss://api.smartdisplay.vzngpt.com
  2. 自动重连 + 心跳保活
  3. 推送类型: device_customization_updated
  4. 推送类型: ai_conversation_config_updated
  5. 推送类型: update / factory_test / reboot

🔴 WebRTC (Realtime 模式)

  1. PeerConnectionFactory + JavaAudioDeviceModule
  2. SDP Offer/Answer 信令交换
  3. ICE Candidate 收集与协商
  4. 双向音频: AudioTrack + AudioSource
  5. DataChannel 用于 sideband 数据
